# Contributing to Pixel Healer

Thanks for your interest in contributing! 🌟

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/pixel-healer.git`
3. Install dependencies: `pnpm install`
4. Start dev server: `pnpm dev`

## Development

### Prerequisites
- Node.js 20+
- pnpm

### Commands
- `pnpm dev` — Start development server
- `pnpm build` — Build for production
- `pnpm preview` — Preview production build
- `pnpm lint` — Run ESLint
- `pnpm test` — Run tests

## Pull Request Process

1. Create a feature branch: `git checkout -b feat/your-feature`
2. Make your changes
3. Ensure tests pass: `pnpm test`
4. Commit with conventional commits: `feat: add cool feature`
5. Push and open a PR

### Commit Convention
We use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` — New feature
- `fix:` — Bug fix
- `docs:` — Documentation only
- `chore:` — Maintenance tasks
- `refactor:` — Code refactoring
- `test:` — Adding tests

## Reporting Issues

- Check existing issues first
- Include browser version and OS
- Provide steps to reproduce
- Attach sample images if relevant (or describe them)

## Code Style

- TypeScript for all code
- React functional components with hooks
- Prefer composition over inheritance
- Keep components small and focused

## Questions?

Open a discussion or reach out via issues.
